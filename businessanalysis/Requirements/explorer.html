Key architectural decisions
Core HR System = single source of truth
Position data (FTE, cost centre, classification, reporting line) auto-populates requisitions, JML triggers, and downstream systems. Managers cannot override at the requisition level without upstream position approval. 292 of 310 requirements reference Core HR System.
Identity Platform = IAM authority (not Workflow Platform)
Workflow Platform orchestrates and captures audit evidence but is not the IAM platform. Identity and access flows Core HR System → Identity Platform → downstream. Workflow Platform's role is orchestration and evidence only (per CHR-433 discussion thread).
Effective-dating is non-negotiable
Every position change, pay movement, increment, restructure, and working arrangement must be effective-dated. Future and back-dated changes must be supported without inconsistency across payroll and access. Present across all 12 approved journeys.
Document Management System / Word / Excel → retire
COI forms, screening templates, interview guides, reference check evidence, background check citations currently in Document Management System as Word/PDF. Every approved journey moves these in-system to Core HR System. 24 requirements explicitly reference legacy document management retirement.
